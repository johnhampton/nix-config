- [ ] **Step 2: Write the script**

Write the following to `$TAP/bin/add-downgrade`:

```bash
#!/usr/bin/env bash
# add-downgrade — fetch a historical cask .rb from homebrew-cask, rename it
#                 to <base>@<version>, and drop it into this tap.
#
# Usage: add-downgrade <cask-name> <version>
#   e.g. add-downgrade claude-code@latest 2.1.119
#        add-downgrade visual-studio-code 1.95.0

set -euo pipefail

usage() {
  cat <<USAGE >&2
usage: add-downgrade <cask-name> <version>
  e.g. add-downgrade claude-code@latest 2.1.119
       add-downgrade visual-studio-code 1.95.0
USAGE
  exit 1
}

[[ $# -eq 2 ]] || usage

CASK="$1"
VERSION="$2"

[[ -n "$CASK" ]] || { echo "error: cask name is empty" >&2; exit 1; }
if [[ -z "$VERSION" || "$VERSION" =~ [[:space:]] ]]; then
  echo "error: version is empty or contains whitespace" >&2
  exit 1
fi

# Repo root is the parent of this script's directory (script lives at <repo>/bin/).
REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"

# First letter of the cask name is the homebrew-cask shard.
LETTER="${CASK:0:1}"

# Base name = everything before the first '@' (or the whole name if no '@').
BASE="${CASK%%@*}"

API_PATH="Casks/${LETTER}/${CASK}.rb"
EXPECTED_MSG="${CASK} ${VERSION}"

# Find the commit on homebrew-cask whose first message line is exactly
# "<cask> <version>".  homebrew-cask commits one cask version per commit
# with that message format.
SHA="$(EXPECTED_MSG="$EXPECTED_MSG" gh api \
  -X GET "/repos/Homebrew/homebrew-cask/commits" \
  -f "path=${API_PATH}" \
  -f "per_page=50" \
  --jq '.[] | select(.commit.message | split("\n")[0] == env.EXPECTED_MSG) | .sha' \
  | head -n 1)"

if [[ -z "$SHA" ]]; then
  echo "error: version ${VERSION} not found in last 50 commits for ${CASK}" >&2
  echo "       check the cask name, version, or increase per_page in the script" >&2
  exit 1
fi

OUT_DIR="${REPO_ROOT}/Casks/${LETTER}"
OUT_FILE="${OUT_DIR}/${BASE}@${VERSION}.rb"

if [[ -e "$OUT_FILE" ]]; then
  echo "error: ${OUT_FILE} already exists" >&2
  exit 1
fi

mkdir -p "$OUT_DIR"

RAW_URL="https://raw.githubusercontent.com/Homebrew/homebrew-cask/${SHA}/Casks/${LETTER}/${CASK}.rb"

curl -fsSL "$RAW_URL" \
  | sed "s/cask \"${CASK}\"/cask \"${BASE}@${VERSION}\"/" \
  > "$OUT_FILE"

cat <<EOF

✓ wrote ${OUT_FILE}
  source commit: ${SHA}

next:
  cd "\$(brew --repo johnhampton/downgrades)"
  git add Casks/${LETTER}/${BASE}@${VERSION}.rb
  git commit -m "add ${BASE}@${VERSION}"
  git push

then in darwin/homebrew.nix replace
  "${CASK}"
with
  "johnhampton/downgrades/${BASE}@${VERSION}"
and run: just switch
EOF
```

**Files:**

- [ ] **Step 1: Write the README**

Write the following to `$TAP/README.md`:

```markdown
# homebrew-downgrades

Personal Homebrew tap for pinning casks to specific older versions.

Homebrew removed URL-based and bare-file cask installs in 2025, so the supported way to install a previous cask version is via a tap. This tap holds renamed copies of historical cask `.rb` files: each entry is named `<base>@<version>` (with any pre-existing `@`-suffix on the original cask stripped) so it has no counterpart in the official `homebrew-cask` tap, blocking accidental `brew upgrade` collisions.

## Quick start

```sh
brew tap johnhampton/downgrades
brew install --cask johnhampton/downgrades/<base>@<version>
```

## Pinned casks

| Cask | Version | Pinned |
| --- | --- | --- |
| `claude-code@2.1.119` | 2.1.119 | 2026-04-27 |

## Adding a new pinned version

```sh
cd "$(brew --repo johnhampton/downgrades)"
bin/add-downgrade <cask-name> <version>
# review the generated file, then:
git add Casks/<letter>/<base>@<version>.rb
git commit -m "add <base>@<version>"
git push
```

The script:

1. Finds the homebrew-cask commit whose first-line message is `<cask-name> <version>`.
2. Pulls the raw `.rb` at that commit.
3. Strips any `@`-suffix from the cask name to compute `<base>`, then renames `cask "<cask-name>"` → `cask "<base>@<version>"`.
4. Writes to `Casks/<letter>/<base>@<version>.rb` (where `<letter>` is the first letter of the original cask name).

## Caveats

- No auto-upgrade. Pinned versions stay pinned until you swap them.
- Manual maintenance: each new pin is a manual API lookup + commit.
- Renamed casks intentionally break `brew upgrade` for that cask — the renamed cask has no counterpart in the official tap to upgrade against.
```
